2. BACKEND SETUP
- Go to backend/
- IMPORTANT: Use Python 3.12 (not 3.13). Python 3.13 has compatibility issues with numpy 1.26.4.
  Check your version: python --version
  If using py launcher on Windows: py -3.12 -m venv .venv
- Create a virtual environment:
  python -m venv .venv
  On Windows PowerShell: .\.venv\Scripts\Activate.ps1
  On Windows CMD: .venv\Scripts\activate
  On Mac/Linux: source .venv/bin/activate
- Install dependencies:
  pip install -r requirements.txt
- Optional quick verification that the shipped CNN artifact can be loaded:
  python check_real_model.py
  Expected output starts with: REAL_MODEL_LOAD_OK
- Start the API:
  uvicorn app.main:app --reload --host 0.0.0.0 --port 8000
  Note: If port 8000 is already in use, use a different port (e.g., --port 8001)
        and update frontend/.env.local accordingly.

3. FRONTEND SETUP
- Go to frontend/
- Create .env.local with the backend API URL:
  NEXT_PUBLIC_API_BASE_URL=http://127.0.0.1:8000
  NEXT_PUBLIC_API_URL=http://127.0.0.1:8000
  (Change port if backend is running on a different port)
- Install dependencies:
  npm install
- Start frontend:
  npm run dev
- Frontend runs at http://localhost:3000

4. REAL MODEL NOTE
- backend/.env currently enables the shipped pretrained CNN:
  ENABLE_REAL_MODEL=1
- The first real inference can take noticeably longer on first run while Keras/JAX warms up the model.
- If your machine cannot run the shipped model path cleanly, temporarily set ENABLE_REAL_MODEL=0 to use the fallback route for debugging only.
